This page contains a short description of various terms used throughout this documentation.

??? abstract "Ceph"

    A reliable and scalable storage solution for object-, block- ,and file storage on commodity hardware.

??? abstract "Cilium"

    The default CNI in many managed Kubernetes offerings available.

??? abstract "Container Network Interface (CNI)"

    CNI plugins implement the Kubernetes network model and provide functionality such as pod-to-pod communication
    or network policies.

??? abstract "GitOps"

    The practice of handling one or more Git repositories as the single source of truth for declarative and auditable application and/or infrastructure deployments.

??? abstract "Kubernetes"

    A container orchestrator originally developed by Google.

??? abstract "Open Container Initiative (OCI)"

    The [Open Container Initiative (OCI)](https://opencontainers.org/) defines open standards regarding container images and runtimes.

??? abstract "PersistentVolumes"

    The Kubernetes representation of a piece of storage provisioned by a storage volume plugin or an administrator.

??? abstract "Rook"

    A Kubernetes operator for Ceph, providing file-, block-, and object storage on top of Kubernetes.

??? abstract "Single Board Computer (SBC)"

    A very small computer that normally fits in your hand. Often compromises on performance. Often very power-efficient due to ARM architecture.

??? abstract "Talos"

    A minimal, immutable, declarative operating system tailor-made for running Kubernetes.

??? abstract "Turing RK1"

    A powerful Single Board Computer (SBC) manufactured by TuringPi.
